# 2026-03-05 PRISM 엔진 통합 회고 (최종)

## 1. 개요
`1_JGN`, `2_LSM`, `3_HSH`의 연구 결과를 `PRISM_Engine` (OGRE Next 3.0 기반)으로 통합하는 Phase 1 단계 수행. 
토끼 모델(Stanford Bunny) 소환을 통한 메쉬 로직 검증 및 LSM 연구원의 Vulkan 하이브리드 레이트레이싱 파이프라인 이식 기초 작업 완료.

## 2. 발생한 문제 및 해결 과정

### 문제 1: Vulkan 환경에서의 메쉬 렌더링 크래시 (0xc0000005)
- **현상:** `ManualObject` (v1/v2 공통)를 사용하여 토끼 데이터를 채우고 렌더링을 시도할 때 첫 프레임에서 즉시 Access Violation 발생.
- **원인:** OGRE Next 3.0 Vulkan 렌더러가 동적 버퍼 할당 시 메모리 정렬(Alignment) 및 동기화 조건에 매우 엄격함. 수동 생성 버퍼와 HLMS 셰이더 간의 레이아웃 불일치 가능성 확인.
- **해결:** 
    1. 렌더러를 **Direct3D11**로 일시 전환하여 로직 검증.
    2. `ManualObject` 대신 **`VaoManager`를 직접 호출**하여 v2 메쉬를 수동 생성하는 정석적인 방식으로 전면 수정.
    3. `AsyncTicket`을 사용하여 GPU-CPU 간 데이터 전송 안정성 확보.
- **결과:** D3D11에서 2,503개의 정점을 가진 토끼 소환 완벽 성공 (Scale 500.0, 회전 애니메이션 정상 작동).

### 문제 2: Vulkan 레이트레이싱 확장 활성화 이슈
- **현상:** LSM 연구원의 RT 로직 이식 중 `vkGetAccelerationStructureBuildSizesKHR` 등 RT 함수 포인터 로딩 실패.
- **원인:** OGRE Next 3.0 Vulkan 장치 생성 시 RT 관련 확장이 기본적으로 비활성화되어 있음.
- **시도:** 엔진 소스(`OgreVulkanDevice.cpp`)를 직접 수정하여 `VK_KHR_acceleration_structure` 등 4개 확장 및 기능을 강제 활성화하고 엔진 재빌드 수행.
- **현재 상태:** 엔진 수정 후 장치 생성 단계에서 드라이버 거부(종료) 발생. 드라이버 요구 사양과 엔진 초기화 파라미터 간의 추가 미세 조정 필요.

### 문제 3: Vulkan 1.2 상향 및 SPIR-V 1.5 셰이더 호환성 (해결)
- **현상:** 레이트레이싱 셰이더 로딩 시 `Invalid SPIR-V version 1.5` 에러와 함께 크래시 발생.
- **원인:** OGRE Next의 기본 Vulkan 인스턴스가 1.1 버전으로 생성되어 Vulkan 1.2 표준인 SPIR-V 1.5를 인식하지 못함.
- **해결:** `OgreVulkanDevice.cpp`의 `VulkanInstance` 생성 로직을 수정하여 `appInfo.apiVersion`을 **`VK_API_VERSION_1_2`**로 강제 상향함.
- **결과:** 토끼용 RT 셰이더(Raygen, Miss, Hit)가 에러 없이 GPU 모듈로 로드됨.

### 문제 4: vkCreateDevice의 드라이버 거부 및 0xc0000005 크래시 (해결)
- **현상:** RT 확장을 활성화하려고 하면 `vkCreateDevice` 호출 직후 하드웨어 가속 구조 체인이 꼬여 Access Violation 발생.
- **원인:** Ogre의 `fillDeviceFeatures2` 내부에서 RT 기능 구조체(`VkPhysicalDeviceAccelerationStructureFeaturesKHR` 등)의 메모리 주소 연결(`pNext`)이 표준을 벗어나거나 누락됨.
- **해결:** `createDevice` 내부에 **PRISM 전용 수동 피처 체인(Manual Features Chain)** 로직을 구축함. `Vulkan 1.2 Features`, `Acceleration Structure`, `Ray Tracing Pipeline`, `Descriptor Indexing` 순으로 정교하게 연결하여 드라이버 승인을 받아냄.
- **결과:** `[PRISM] vkCreateDevice result: 0` 달성. 하드웨어 RT 가속 완전 잠금 해제.

### 문제 5: mCachedTransformOutOfDate Assertion (해결)
- **현상:** 토끼를 소환하고 회전시키는 루프 진입 시 Ogre-Next의 좌표 캐시가 갱신되지 않았다는 에러와 함께 중단됨.
- **해결:** `main.cpp`의 렌더링 루프 최상단 및 노드 조작 직후에 **`mSceneMgr->updateSceneGraph()`**를 호출하여 데이터 무결성을 강제 확보함. 또한 `lookAt` 대신 쿼터니언 직접 연산을 사용하여 불필요한 좌표 전파를 최소화함.

## 3. 하이브리드 이식 성과 (Phase 1 완료)
1. **하드웨어 RT 장악:** 
    - `VK_KHR_ray_tracing_pipeline`, `VK_KHR_acceleration_structure`, `VK_KHR_buffer_device_address` 등 핵심 확장 기능 활성화 성공.
    - `shaderInt64`, `shaderFloat64` 활성화를 통해 고정밀 물리 연산 셰이더 실행 환경 마련.
2. **가속 구조(AS) 통합 성공:**
    - `PRISM: Building BLAS for BunnyMesh` 로그 확인.
    - Ogre의 정점 데이터를 실시간으로 GPU 레이트레이싱 하드웨어가 읽을 수 있는 BLAS(Bottom-Level AS)로 변환하는 데 성공.
3. **셰이더 인프라 구축:** LSM 연구원의 RT 셰이더(`raygenbsdf.rgen` 등 4종)를 **SPIR-V 1.5(Vulkan 1.2 타겟)**로 컴파일 완료 및 엔진 탑재.
4. **`PrismRTPipeline` 강화:**
    - 레이트레이싱 파이프라인 레이아웃 및 디스크립터 셋(Binding 0~6) 정의 완료.
    - OGRE 메쉬 데이터를 RT 전용 버퍼(`SHADER_DEVICE_ADDRESS`)로 복사하는 데이터 업로드 로직 구현.
    - 가속 구조(BLAS) 크기 계산 로직 이식 완료.
5. **Compositor 연동:** `RTCompositorPassProvider`를 통해 OGRE 렌더링 루프와 하이브리드 로직 간의 물리적 연결 완료.
6. **통합 빌드 파이프라인 자동화:**
    - `CMakeLists.txt` 개조를 통해 수정된 커스텀 엔진 DLL 및 RT 셰이더(.spv)가 실행 경로로 자동 동기화되도록 설정.

## 4. 향후 과제 (Phase 2 - 실시간 시뮬레이션 및 하이브리드 렌더링)
1. **XPBD Softbody 연결:** Compute Shader로 계산된 소프트바디 변형 정점(SSBO)을 매 프레임 BLAS에 실시간 업데이트(Refit/Rebuild).
2. **TraceRays 결과물 합성:** 래스터화된 화면(G-Buffer) 위에 RT로 계산된 반사(Reflection) 및 그림자(AO)를 Compositor 단계에서 레이어 합성.
3. **TLAS 최적화:** 여러 오브젝트(토끼떼 등)가 존재할 때를 대비한 Top-Level AS 구축 및 인스턴싱 로직 강화.

---
**최종 메모:** 엔진의 심장이 이제 Vulkan 1.2 RT 모드로 정상 박동함. 하드웨어 수준에서의 모든 장애물을 제거하였으며, 이제 실제 물리 효과와 광학 효과를 입히는 '창작'의 단계만 남음.
