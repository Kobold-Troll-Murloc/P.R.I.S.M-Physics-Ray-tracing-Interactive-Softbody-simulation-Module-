## 2. 상세 구현 기술 (Technical Deep Dive)

### A. Disney BSDF & Path Tracing (Shader)
- **알고리즘**: 몬테카를로 샘플링을 통한 최대 7회 바운스의 패스 트레이싱.
- **재질 모델**: `Disney BRDF` (Diffuse, Specular) + `BTDF` (Transmission/Refraction).
  - GGX 분포 함수를 사용하여 금속 및 비금속의 거칠기(Roughness) 표현.
  - Fresnel Dielectric 연산으로 유리의 굴절 및 반사율 제어.
- **지오메트리 직접 참조**: `GL_EXT_buffer_reference2`를 사용하여 셰이더가 GPU 주소(uint64_t)로 직접 정점/인덱스 버퍼를 읽어 법선 벡터 보간(Normal Interpolation) 수행.

### B. 프레임 누적 및 톤매핑 (Progressive Rendering)
- **누적 버퍼**: HDR(RGBA32F) 버퍼인 `mAccumImage`를 생성하여 매 프레임 결과물을 누적.
- **궤적 현상(Ghosting) 해결**: 
  - 카메라 이동 시 `frameCount`를 0으로 리셋하여 누적 버퍼를 덮어쓰도록 구현.
  - 정지 시에만 `1 / (frameCount + 1)` 가중치로 부드럽게 병합.
- **포스트 프로세싱**: Reinhard 기반의 톤매핑과 노출(Exposure) 보정, 감마 교정(2.2)을 셰이더 내에서 일괄 처리.

## 3. 시행착오 및 해결 (Troubleshooting)

### 이슈 1: 카메라 이동 시 토끼 잔상이 남는 문제
- **원인**: 이전 프레임의 이미지를 배경으로 로드하면서 위치가 바뀐 토끼 정보가 계속 남아 피드백 루프 발생.
- **해결**: 레이 미스(Miss) 시 이전 이미지를 읽지 않고 `skyColor`를 사용하도록 수정하고, 이동 감지 시 `frameCount`를 강제 리셋함.

### 이슈 2: 전체적인 화면이 너무 어두움
- **해결**: 
  1. C++ 코드에서 조명 강도(Intensity)를 1.0에서 5.0으로 상향.
  2. 셰이더의 노출 지수(Exposure)를 1.0에서 2.5로 상향 조정하여 전반적인 조도 확보.

### 이슈 3: OgreNode.h 680번 줄 Assertion Error
- **에러**: `!mCachedTransformOutOfDate` (노드 행렬 캐시 만료).
- **원인**: 노드 위치 수정 직후 `updateSceneGraph()` 호출 없이 `_getFullTransform()`으로 행렬 값을 읽으려 함.
- **해결**: Ogre-Next의 규칙에 따라 배치 완료 후 반드시 `mSceneMgr->updateSceneGraph()`를 호출한 뒤 행렬을 획득하도록 구조 변경.

## 4. Ogre-Next 3.0 개발 팁 (Lessons Learned)
1. **헤더 파일 변화**: `OgreRenderWindow.h`는 구버전이며, 최신 버전에서는 `OgreWindow.h`를 사용해야 함.
2. **Item 인스턴스**: 하나의 `Item`은 하나의 `SceneNode`에만 부착 가능. 같은 메시를 여러 노드에 띄우려면 각각 별도의 `Item`을 생성해야 함.
3. **Vulkan 함수 로딩**: RT 확장 기능은 디바이스 수준에서 함수 포인터를 동적으로 로드해야 하며, 드라이버 호환성(Vulkan 1.2+) 체크가 필수임.
