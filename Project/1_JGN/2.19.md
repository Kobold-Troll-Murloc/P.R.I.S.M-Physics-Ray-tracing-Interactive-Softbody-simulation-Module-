GPU와 CPU 간 데이터 동기화를 효율적으로 구현하기 위해서는 다음과 같은 방법들이 활용됩니다:

### 1. 비동기(Asynchronous) 데이터 전송
- CPU와 GPU가 동시에 작업을 수행할 수 있도록, 데이터 전송을 비동기적으로 처리합니다.
- 예: OpenGL의 Pixel Buffer Object(PBO), Vulkan의 비동기 커맨드 큐 등

### 2. 더블 버퍼링/트리플 버퍼링
- 두 개 이상의 버퍼를 번갈아 사용하여, 한쪽에서는 GPU가 데이터를 처리하고 다른 쪽에서는 CPU가 데이터를 준비할 수 있게 합니다.

### 3. 최소한의 데이터 전송
- 매 프레임마다 전체 데이터를 동기화하지 않고, 변경된 데이터(Dirty Flag 방식)만 전송하여 오버헤드를 줄입니다.

### 4. 동기화 객체(Synchronization Primitives) 활용
- 세마포어, 펜스(fence), 이벤트 등 동기화 객체를 사용해 CPU와 GPU의 작업 완료 시점을 명확히 관리합니다.
- Vulkan, DirectX 등에서는 펜스와 세마포어를 적극적으로 사용합니다.

### 5. 맵/언맵(Map/Unmap) 패턴
- GPU 버퍼를 직접 매핑하여 CPU가 데이터를 쓸 수 있게 하고, 작업이 끝나면 언매핑하여 GPU가 사용할 수 있도록 합니다.

### 6. Persistent Mapping
- 버퍼를 지속적으로 매핑해두고, 필요할 때마다 CPU가 데이터를 업데이트할 수 있도록 하여, 매번 매핑/언매핑하는 오버헤드를 줄입니다.

### 7. 동기화 최소화
- CPU와 GPU가 서로를 기다리는 시간을 최소화하도록, 작업 스케줄링과 데이터 흐름을 최적화합니다.

이러한 방법들을 적절히 조합하면, GPU/CPU 간 데이터 동기화의 효율성을 높이고 병목 현상을 줄일 수 있습니다.
GPU/CPU 동기화는 시스템 성능에 중요한 영향을 미칩니다. 동기화가 잘못 설계되면 병목 현상이 발생하여 전체 처리 속도가 저하될 수 있습니다.

### 성능에 미치는 주요 영향

- **병목 현상(Bottleneck) 유발**: CPU와 GPU가 서로의 작업 완료를 기다리게 되면, 한쪽이 유휴 상태(idle)가 되어 자원을 비효율적으로 사용하게 됩니다.
- **프레임 지연(Frame Latency) 증가**: 동기화 지점에서 대기 시간이 발생하면, 렌더링 프레임이 늦어져 사용자 경험이 저하될 수 있습니다.
- **자원 활용률 저하**: 동기화가 잦으면 CPU와 GPU 모두 최대 성능을 발휘하지 못하고, 전체 시스템의 처리량(throughput)이 감소합니다.
- **데이터 일관성 보장**: 반면, 적절한 동기화는 데이터의 일관성과 정확성을 보장하므로, 무조건 동기화를 피하는 것도 바람직하지 않습니다.

### 최적화 방안
- 동기화 지점을 최소화하고, 비동기 처리를 적극 활용하여 CPU와 GPU가 동시에 작업할 수 있도록 설계하는 것이 중요합니다.
- Dirty Flag, 더블/트리플 버퍼링, 비동기 커맨드 큐 등 다양한 기법을 활용해 병목을 줄이고 성능을 극대화할 수 있습니다.

**결론적으로, 동기화는 필수적이지만 과도하거나 불필요한 동기화는 성능 저하의 주요 원인이 되므로, 효율적인 동기화 전략이 필요합니다.**


1-1. 메테리얼과 오버넥스트의 기초 이해
- 메테리얼은 큰 구조체를 의미하며, 오버넥스트는 편리한 기본값을 제공함
- 메테리얼은 조절 가능한 값들을 지니고 있으며, 오버넥스트는 파라미터들을 조정할 수 있음
- 메테리얼은 제재 값이나 텍스처 값 등이 하나의 큰 구조체로 통합되어 있음
- 메테리얼의 기본값은 세팅되는 값들을 사용하며, 오버넥스트는 세팅에 따라 셰이더를 나눔

1-2. 오버넥스트의 특징과 이점
- 오버넥스트는 필요한 기능만 선택적으로 활성화하며, 불필요한 연산을 줄임
- (중요) 기능 선택을 통해 불필요한 연산을 줄이고, 연산의 효율성을 향상시킴
- 이를 통해 유니티 내에서 원하지 않는 기능을 제거하고, 필요한 기능만 선택하여 사용할 수 있음
- 다만, 이를 통해 복잡성 또한 증가하므로, 적절한 조정이 필요함

1-3. 모듈화의 이점과 필요성
- 모듈화를 통해 여러 상황에 대응 가능하며, 시뮬레이션의 정확성을 보장함
- (중요) 이로 인해 코드의 복잡성이 증가하나, 이는 셰이더의 유연성과 호환성을 향상시킴
- 그렇기 때문에 모듈화를 고려하는 것은 중요한 전략이며, 이를 통해 코드의 효율성과 안정성을 모두 잡을 수 있음
- 이는 오버넥스트의 한계를 극복하고, 복잡한 시나리오에 대응하는 데 도움을 줌
